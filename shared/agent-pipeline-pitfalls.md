# Типичные сложности пайплайна Nero Network Office Page

Документ для **всех агентов** и **директора**: что уже ломалось на практике и как **не тратить итерации** на одно и то же. Читать **перед** страницей целиком или при странном поведении на проде.

---

## 1. Тема WordPress Configured WordPress Theme (не вина «контента»)

| Симптом | Причина | Что делать |
|--------|---------|------------|
| **Светлая полоса** между шапкой и тёмным/цветным hero | В `_single.scss` темы у `.site-main` задан **`padding-top: 120px`** (отступ под фиксированный header). Фон там — цвет фона контента (**светлый**). | В **каждом** шаблоне `page-{slug}.php` с полноэкранным hero добавлять сброс, как в **`page-vibecoding.php`**: селекторы `#primary, .site-main, .site-content, #content, .content-area { padding-top: 0 !important; margin-top: 0 !important; }` и при необходимости `style="padding-top:0"` на `<main id="primary">`. **Наташа** включает это в общий `<style>` страницы, если hero первый блок. |
| Контент «уезжает» вниз на обычных страницах | То же правило не трогать глобально в `style.css` темы — только в шаблонах лонгридов с hero. | Локально в PHP-шаблоне страницы. |

## 2. Директор и файл обмена

| Сложность | Как избежать |
|-----------|----------------|
| Субагенты читают **старый** handoff | Перед новой страницей **полная перезапись** `nero-network-handoff.md` одной строкой (`Write`), не `apply_patch` для «очистки». |
| Параллельная пара **перетёрла** handoff (например, Коля\|\|Артём) | Новый протокол: параллельные агенты **не пишут напрямую** в `nero-network-handoff.md`. Они пишут отдельные фрагменты в `.cursor/nero-network-fragments/` (`kolya.md`, `artyom.md`, `alina.md`, `boris.md`, `qa.md`, `lenya.md`), а Директор после пары сам склеивает их в handoff. |
| Агент в параллельной паре переписал **весь** файл вместо своей секции | В промптах параллельным агентам явно писать: **«результат только во фрагмент, не в handoff»**. После пары директор читает фрагменты, проверяет маркеры и переносит их в handoff в фиксированном порядке. |
| Дубли блоков `=== МАКС (QA) ===` / `=== ЛЁНЯ` / `=== ЮРА` | Перед финалом Директор проверяет, что одноимённый маркер встречается **ровно один раз**. Если агент писал напрямую и сделал дубль — оставить один актуальный блок или дозапросить отчёт во фрагмент. |
| Юра пишет «опубликовано», но нет блока `=== ЮРА (ПУБЛИКАЦИЯ) ===` | До QA/Лёни Директор обязан проверить не только live HTML, но и наличие блока Юры в handoff. Без блока Юры публикация считается незавершённой: дозапустить Юру только на запись отчёта/журналов. |
| **Директор начал делать роли сам** | В Cursor Cloud текущий агент уже Директор-оркестратор. `director` не запускается как отдельный субагент; нужно запускать только ролевые Task/generalPurpose. Лонгрид, hero, Борис, Наташа, Артур, Юра, QA и SEO-ядро самому делать нельзя. |
| Разные агенты смотрят разные журналы опубликованных страниц | Канонический путь published-pages: `<PROJECT_ROOT>/nero-network-office-page/shared/published-pages.md`. Ledger Кирилла: `<PROJECT_ROOT>/shared/kirill-news-ledger.md`. В промптах использовать переносимые пути от `<PROJECT_ROOT>`, не абсолютные локальные пути и не неоднозначное `shared/published-pages.md`. |
| **Task(`lenya`)** в Cursor | Тип субагента **`lenya`** в списке Task **может отсутствовать**. Вызывать **`generalPurpose`** с промптом по skill **seo-auditor-lenya** (IndexLift + handoff). |
| **Task(`boris`)** в Cursor | Тип **`boris`** может отсутствовать в списке Task. Вызывать **`generalPurpose`** + skill **animator-boris** (тот же промпт, что в **director-nero-network**). |
| Дублирование этапов | Промпты из **director-nero-network** копировать дословно: **Алина и Борис параллельно**; у Алины — только hero; у Наташи — вставить блок Бориса + не затемнять hero + сброс padding при необходимости. |
| Ускорение пайплайна | После **Юры** запускать **параллельно** **Макс (qa)** и **Лёня** — оба смотрят один URL, независимы. Не выстраивать их в очередь. |
| Что **нельзя** параллелить без поломки | **Женя** до готовности Коли **и** Артёма; **Артур** до лонгрида; **Наташа** до hero+Бориса+текста; **Юра** до HTML от Наташи. |

**Итого три параллельные пары:** Коля||Артём → … → Алина||Борис → … → Макс||Лёня.

## 3. SEO Коля

| Сложность | Как избежать |
|-----------|----------------|
| Нулевая частота у **нового** продукта (например `kimi k2.6`) | Нормально: помечать как **прогнозный/низкочастотный** кластер, опираться на ближайшие семена (нейросеть для кода, kimi ai). |
| Мало вызовов Wordstat | Skill требует **~10+** запросов — не сокращать без причины. |

## 4. Артём (research)

| Сложность | Как избежать |
|-----------|----------------|
| **Галлюцинации** цен, дат релиза, бенчмарков | В handoff явно разделять: **подтверждено** / **не подтверждено** / **слух**; Женя не должна подавать сомнительное как факт. |

## 5. Женя (лонгрид)

| Сложность | Как избежать |
|-----------|----------------|
| Короткий текст при ТЗ «лонгрид» | Ориентир **8k+** знаков из **knowledge-base**; если не выходит — указать причину в handoff, не молча. |

---

## 6. Алина (hero Canvas)

| Сложность | Как избежать |
|-----------|----------------|
| Визуально **как вайбкодинг**, только другие слова | Skill **animator-alina**: эталон — **каркас движка**, не макет; обязательны **новая сцена**, **новые фазы цикла**, **чеклист отличий** (≥6 строк). |
| **Тёмный** full-screen hero без запроса | По умолчанию — **светлый фон + тёмная типографика**; тёмный hero — только если владелец **явно** попросил. |
| Стили hero не подхватились | Весь CSS hero — **inline** в `<style>` в секции hero; не полагаться на `style.css` темы. |

---

## 6а. Борис (визуальный блок в статье, не hero)

| Сложность | Как избежать |
|-----------|----------------|
| Спутали с hero | Борис — **не** первый экран; **не** `100vh` как у hero; вставка **в теле** лонгрида (часто после 2-го H2). Skill **animator-boris**. |
| Два одинаковых `id` у canvas | Алина и Борис работают **параллельно** — **разные** `id` для canvas и **разные** имена файлов движка. |
| Борис повторяет сцену Алины | Выбрать режим **продолжение** или **контраст**; не копировать центральный объект и цикл hero. |
| Блок Бориса выглядит «криво» — узкая колонка, canvas по центру | Skill **animator-boris**: обязательны **сплит/сетка/карта**, eyebrow + мини-заголовок, не сдавать только центрированный виджет. |
| Наташа забыла вставить блок | В handoff искать `=== БОРИС (БЛОК СТАТЬИ, НЕ HERO) ===` и якорь вставки. |

## 7. Наташа (вёрстка страницы)

| Сложность | Как избежать |
|-----------|----------------|
| Белая полоса под шапкой | См. §1 — **сброс padding-top** для `#primary` / `.site-main` в `<style>` шаблона. |
| Hero стал **тёмным** после сборки | Не переопределять фон/текст hero глобальными правилами `{slug}-page`; hero — зона Алины. |
| Сломанные `<script>` / `<canvas>` | Публиковать как **PHP в тему** (Юра), не через WP REST с «очисткой» HTML. |
| Пропал **блок Бориса** или второй canvas | Проверить порядок: hero → контент → **Борис** по якорю; внизу страницы — **два** script-движка (hero + Борис), если оба блока есть. |
| **Второй блок** (введение сразу после hero) — центр / нет ритма | Skill **designer-natasha**: текст вступления **слева**, сетка «текст + декор», акцент слева; **не** только два центрированных абзаца. См. **`longread-page-design-system.md`** §«Введение сразу после hero». |
| После FTP **не видно правок** на сайте | **`wp cache flush`**, проверка с **Ctrl+F5**; кэш плагина/хостинга. |

### 7а. Второй блок после hero (введение)

| Симптом | Что делать |
|---------|------------|
| Вступление по центру, без декора | Верстать по §«Введение сразу после hero» в **longread-page-design-system.md**; при необходимости `text-align: left !important`. |
| Схлопнутая высота / «белый» hero | Не использовать Tailwind-only классы без своего CSS; hero — явный `min-height: 100vh` + `position: relative` в inline-стилях (см. §6 Алина). |
| Правое выравнивание текста | По умолчанию для Configured WordPress Theme — **левое**; иначе только по явному ТЗ владельца. |
| Нет **nero-ai-header** / пункты ведут на главную | Перед hero: `require` `partials/nero-ai-longread-bootstrap.php` + `$nero_header_nav_links` по якорям страницы + `partials/nero-ai-site-header.php`. См. commit `74b3c9f`. |

### 7б. Двойной скролл

| Симптом | Что делать |
|---------|------------|
| Два вертикальных скроллбара (body + `#primary`) | **Не** ставить `overflow-x: hidden` на `.nero-ai-home-page` / `#primary` — браузер включает `overflow-y: auto`. Вместо этого: `html { overflow-x: clip; }`, у Kadence-обёрток `overflow: visible !important; height: auto`, у `.nero-ai-home-page` — `overflow: visible; min-height: 0`. Подключить `nero-ai-longread-bootstrap.php` и `assets/css/nero-ai-longread-ui-compat.css`. См. `shared/nero-ai-longread-ui-compat.css`, commit `28a19bb`. |

## 8. Юра (публикация)

| Сложность | Как избежать |
|-----------|----------------|
| **WP API** ломает скрипты | Для страниц с canvas/script — только SSH/SCP/SFTP/FTP как `page-{slug}.php` в тему; в Cloud сначала SSH/SCP/SFTP; см. skill **publisher-yura**. |
| Ошибки **WP CLI** из-за кавычек в excerpt | Экранировать кавычки в `--post_excerpt` при создании/обновлении страницы. |
| Пустой **meta description** | `post_excerpt` недостаточно — тема может не выводить excerpt в `<head>`. **Наташа** обязана добавить в шаблон `page-{slug}.php` **до `get_header()`**: `document_title_parts` + `wp_head` с `<meta name="description">` и `og:*` из Description handoff. **Юра** после публикации проверяет наличие meta description в live HTML. |
| **SSH_HOST / FTP_HOST = IP панели** | IP хостинга часто **не принимает** 21/22 снаружи. Использовать **домен сайта** (`WP_SITE_URL` без схемы). Пути: `REMOTE_SITE_ROOT` = runtime home SFTP-пользователя (например `/home/d/user/site.ru/public_html`), не устаревший `/home/devham/public_html`. |
| `SSH_THEME_PATH` или `REMOTE_WP_THEMES` указывает не туда | Env-путь — **не источник истины**. Перед upload Юра обязан получить runtime-путь активной темы через WordPress: `wp eval 'echo get_stylesheet_directory();'`. Если env отличается от runtime-пути, грузить в `get_stylesheet_directory()` и записать расхождение в отчёт. |
| Файл шаблона загружен, но прод всё равно рендерит **`page.php`** | Это **не успех публикации**. Сначала определить **активную тему** (`stylesheet` / `template`) и **runtime-путь** (`get_stylesheet_directory()`), затем грузить `page-{slug}.php` именно туда. После публикации проверить **живой HTML**, а не только HTTP 200: должны быть уникальные маркеры кастомного шаблона (`{slug}-page`, `canvas id`, класс hero). Если видишь в HTML `<div class="container"><nav class="breadcrumbs">` и пустой `entry-content`, значит отдается дефолтный `page.php`. |
| Файл лежит в правильном каталоге, но WordPress не может его подключить | Проверить права: файл шаблона `644`, каталог темы и родительские каталоги до `wp-content/themes` доступны на обход (`755` или рабочий эквивалент хостинга). |
| WP CLI падает из-за версии PHP на хостинге | Не бросать публикацию и не переходить на WP API. Использовать рабочую связку PHP/WP-CLI, например `/usr/local/bin/php8.1 /usr/local/bin/wp-cli.phar ...`, либо безопасный PHP `eval-file` через SSH. |
| Шаблон лежит в теме, но страница всё равно не берёт его | Проверить три вещи подряд: 1) файл в **активной** теме, 2) при необходимости у страницы выставлен `_wp_page_template = page-{slug}.php`, 3) очищен **не только** объектный кэш WP, но и внешний кэш (например, **WP Fastest Cache** / кэш хостинга). Только после этого снова читать live HTML. |
| Проверка «200 OK» дала ложное чувство успеха | `200` подтверждает только доступность URL. У Юры обязательна вторая проверка: в HTML реально присутствуют hero/`canvas`/уникальные маркеры кастомного шаблона, а не только шапка/футер сайта. |

## 8а. Доступность: `alt` и ссылки (SEO)

| Симптом | Что делать |
|---------|------------|
| Аудит: «у изображений нет `alt`», «текст ссылки неразличим для программ чтения» (часто баннер из AD_BANNER_*) | В шаблоне у **каждого** `<img>` должен быть **`alt`**. Ссылка только с картинкой без `alt` = «немая» ссылка. Использовать шаблон баннера из **advertiser-artur** (`alt` + `rel="noopener noreferrer"` + `target="_blank"`). |
| Дублирование замечаний после публикации | **Артур** выдаёт баннер с `alt`; **Наташа** не вырезает атрибуты; **Макс** проверяет §4.2 в **qa-checker**. |

## 9. Макс (QA)

| Сложность | Как избежать |
|-----------|----------------|
| Пропущен **визуальный зазор** под header | После скриншота верха страницы проверять: **нет ли большой пустой светлой полосы** между шапкой сайта и первым блоком hero/контента; если есть — вернуть на **Наташу/Юру** с отсылкой к сбросу `padding-top` (§1). |
| Не проверен **блок Бориса** | Проскроллить до места вставки; **второй** `<canvas>` (если задуман) и отсутствие ошибок в консоли от движка Бориса. |
| Зазор и хлебные крошки видны, хотя Юра пишет «правки выложены» | Подозревать не только CSS, но и **неприменённый кастомный шаблон**. Если одновременно видны breadcrumbs и нет характерных маркеров hero/контента, сначала проверить, что страница рендерится не через дефолтный `page.php`, а уже потом гонять ещё одну CSS-итерацию. |

## 10. Лёня (SEO-аудит после публикации)

| Сложность | Как избежать |
|-----------|----------------|
| **IndexLift** / краулер не получает HTML (`low confidence`, блокировка хостинга) | Не считать это автоматически «сломанной страницей»; дополнительно проверить curl с User-Agent. Команда: `node scripts/run-audit.js --url "<URL>" --mode single-page --tier basic`. URL **со слешем** и **без слеша** — оба варианта. |
| **Task QA/Лёня прерван** | Директор завершает аудит **сам** (curl + IndexLift + скриншот при возможности), пишет `qa.md`/`lenya.md`, исправляет и **перепубликует** без запроса подтверждения пользователю. |
| Нет типа Task **lenya** | См. §2 — **generalPurpose** + skill **seo-auditor-lenya**. |

## 11. Быстрая связка «симптом → кто правит»

- Полоса под шапкой → **Наташа** (CSS в шаблоне) → **Юра** (перезаливка PHP).
- Полоса под шапкой **вместе** с breadcrumbs и пустым шаблонным контейнером → сначала **Юра** (активная тема / `_wp_page_template` / кэш), потом уже **Наташа** при необходимости.
- Hero не тот цвет / затемнили → **Алина** (источник) или **Наташа** (убрать лишние переопределения).
- **Наложение в hero** (tasks/pills/H1) → **Алина/Наташа**: ≤900px CSS Grid stack; `padding-top` под sticky header; не ставить pills поверх tasks через `top: 280px`.
- Анимация как у вайбкодинга → **Алина** (доработать сцену + чеклист).
- Блок Бориса дублирует hero / не на месте → **Борис** (логика сцены) или **Наташа** (якорь вставки).
- Скрипты на странице текстом → **Юра** (не тот канал публикации или обёртка блока в WP).
- Нет `alt` у баннера рекламный партнёр из env / «немая» ссылка-картинка → **Артур** (шаблон баннера) → **Наташа** (вёрстка) → **Юра** (перезаливка PHP).
