# Эталон hero-блока с анимацией (страница «Вайбкодинг»)

Ориентир: [${PUBLIC_SITE_HOST}/vibecoding/](https://${PUBLIC_SITE_HOST}/vibecoding/) — шаблон **Configured WordPress Theme** (`page-vibecoding.php` + `assets/js/vibecoding-engine.js`).

**Важно для аниматора:** визуал вайбкодинга — это **пример одной реализации**, а не шаблон «поменять только текст». На каждой новой странице — **другая метафора пространства**, другой центральный объект и другой цикл фаз в JS; в ответе — **чеклист отличий** от эталона (см. skill **animator-alina**).

## Разметка hero

### Цвет и контраст (по умолчанию)

- Эталон — **светлый** офис (сетка на светлом фоне). Hero с Canvas **по умолчанию светлый**: фон в духе темы сайта (`#fff` / `#f8fafc` / лёгкая сетка), **тёмные** H1 и подзаголовок (`#0f172a`, приглушённый второй строкой). Панели `vl-ui-tasks` / `vl-ui-pill` — светлые карточки с тёмным текстом.
- **Тёмный** полноэкранный hero (например «ночной терминал») — **только** если в задаче от владельца **явно** запрошено. Иначе аниматор ориентируется на **светлую** гамму и контраст на светлом фоне.

- Секция **`#factory`**, класс **`fullscreen-white-office`**: полноэкранный блок, сетка фона, `100vh`, выход на всю ширину.
- Поверх — **абсолютное позиционирование** текста: **H1** (крупный, двухстрочный акцент), под ним **подзаголовок** (`giant-seo-sub`) — **сразу отвечает на вопрос из заголовка** (GEO/интент первой строки).
- Слева сверху — **CTA** (например Telegram). Справа — панель **`vl-ui-tasks`** («этапы» процесса). Снизу по центру — **`vl-ui-pill`** (метрики-«дашборд»).
- Внутри — **`canvas#vibe-factory-canvas`** + скрипт **`vibecoding-engine.js`** (defer).

## Движок анимации (Canvas 2D)

- Один файл-движок: классы **`Conveyor`** (конвейер), **`WebsiteTerminal`** (окно сайта / ракета / деньги), **`Agent`** (маленькие «люди»).
- **Пять агентов** — одна визуальная система, **разные роли** (`role`):
  - `1_architect` — жёлтый (`agentYellow`);
  - `2_seo` — зелёный (`agentGreen`);
  - `3_coder` — синий (`agentBlue`);
  - `4_designer` — розовый (`agentPink`);
  - `5_deployer` — фиолетовый (`agentPurple`).
- У каждого: позиция, цвет, **`stepTrig`** (фаза цикла 0–200), массив **`dialogs`** для пузырей-реплик.
- Цикл **`frame`**: движение к «центру сцены», объекты окружения, предметы на ленте/пути, **`createBubble`** для сценических подсказок.
- **Новая тема страницы** = **полностью новая сцена по смыслу**: интерьер, объекты, взаимодействия и цикл «что происходит на экране» — **не** перенос завода с ${PUBLIC_SITE_HOST}.

## Эталонный JS (всегда под рукой)

В плагине лежит **полный рабочий пример** движка:

- **`shared/hero-engine-example.js`** — полный **JS** (не JSON), копия `vibecoding-engine.js` с комментарием в шапке. Перед написанием новой страницы **открой этот файл**, скопируй в тему как `имя-темы-engine.js` и правь сцену.

## Что оставляем только «людей» / что переделываем

**Оставляем по типу:** персонажи **`Agent`** (пять фигурок, цвета, роли `1_architect` … `5_deployer`, походка, случайные реплики из `dialogs[]`), общий каркас Canvas (`resize`, `frame`, `engineloop`, отрисовка пузырей).

**Переделываем под тему страницы (обязательно, не косметика):**

- **Объекты и интерьер:** класс **`Conveyor`** (или замени на другой транспорт/фон), класс **`WebsiteTerminal`** (или замени на другой центральный объект — не «окно сайта», если тема не про сайт).
- **Взаимодействия:** что едет по ленте (`itmX1`… и цвета `C.box*`), что рисуется в «терминале» по фазам `buildProgress`, финал цикла (ракета/ZIP/деньги — **заменить** на метафору темы).
- **Тексты:** все **`dialogs`**, все **`createBubble`** в `engineloop`, подписи этапов в HTML (`vl-ui-tasks`), pill — **строго под интент страницы**.
- **Геометрия сцены:** при необходимости `targetX` / `targetY` в `Agent.draw`, позиции `entities`, чтобы персонажи ходили к **новому** объекту.

| Оставить (тип) | Переделать под тему |
|----------------|---------------------|
| Структура hero (fullscreen + H1 + ответ + UI + canvas) | Тексты H1, подзаголовка, этапов, pill, CTA |
| Класс `Agent`, 5 цветов, `role`, пузыри | Реплики `dialogs[]`, тайминги `stepTrig`, цель движения |
| Каркас `engineloop` | Предметы на ленте, фазы `createBubble`, логика центрального объекта |
| — | **`Conveyor` / `WebsiteTerminal`** — новая сцена, новые действия |
