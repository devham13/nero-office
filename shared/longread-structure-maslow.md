# Структура лонгрида: пирамида Маслоу + FAQ + кейсы

Единый ориентир для **Коли** (оглавление), **Жени** (текст), **Наташи** (вёрстка), **Макса** (QA). Перед лонгридом прочитай вместе с `shared/knowledge-base.md`.

## Пирамида потребностей Маслоу на странице

Страница-лонгрид строится **снизу вверх**: сначала закрываем базовые страхи и боль, затем доверие и результат, в конце — рост и решение «взять услугу».

| Уровень Маслоу | Что чувствует ЦА | Блоки страницы (H2) | Задача блока |
| --- | --- | --- | --- |
| **1. Физиологические / выживание бизнеса** | «Теряю заявки», «хаос», «не успеваем отвечать» | Hero + ввод после hero; **проблема и цена бездействия** | Зацепить боль, показать срочность без запугивания |
| **2. Безопасность** | «Не сломают CRM», «152-ФЗ», «бот соврёт», «подрядчик пропадёт» | **Риски, комплаенс, границы AI**, этапы внедрения, SLA | Снять страхи, показать контроль и предсказуемость |
| **3. Принадлежность / социальное доказательство** | «У кого уже работает», «наша отрасль» | **Кейсы и примеры** (без ссылок в тексте), отрасли, сценарии | Доверие через узнаваемые ситуации |
| **4. Уважение / статус и результат** | «Буду выглядеть профессионально», «цифры руководству» | **Результаты, KPI, сравнение с альтернативами**, экспертиза Nero | Показать измеримый эффект и компетенцию |
| **5. Самореализация** | «Вырасту с AI», «масштабирую без найма армии» | **Стратегия, дорожная карта**, масштабирование, финальный оффер + CTA | Связать услугу с ростом бизнеса |

**Порядок H2 в handoff Коли** должен отражать эту лестницу (допускаются уточняющие H3 внутри уровня). **FAQ** — после уровней 4–5, **перед** финальным CTA (или сразу перед блоком «Стоимость / как заказать»).

## Кейсы на странице — без ссылок

В **опубликованном** лонгриде (текст Жени, HTML Наташи):

- **Запрещено:** `<a href="...">`, голые URL (`https://…`), UTM-ссылки, «подробнее на сайте X» с кликабельным адресом.
- **Разрешено:** название компании/источника **текстом** («кейс OpenAI Inbound Sales», «обзор amoCRM SalesBot»), формулировки «по публичным данным вендора», «проектная модель Nero Network».
- Цифры в кейсе — только если есть подтверждение в research; без ссылки на источник в тексте страницы.

**Артём** в handoff ведёт ссылки в отдельном блоке **«Источники для верификации (не для лонгрида)»** — Женя и Наташа **не копируют** его на страницу.

## FAQ — подробный и под нейровыдачу

Минимум для лонгрида офиса:

| Параметр | Ориентир |
| --- | --- |
| Количество вопросов | **12–18** (при узкой теме — не меньше **10**) |
| Формулировка вопроса | Как в голосовом/чат-запросе к ИИ: «Сколько стоит…», «Как внедрить…», «Чем отличается от…», «Нужен ли программист…» |
| Ответ | **80–200 слов**, один прямой ответ в первых 2–3 предложениях, затем детали |
| Ключи | Вплетать фразы из ядра Коли и NЧ/FAQ из Google Таблицы |
| Разметка | H2 «Частые вопросы о …»; каждый вопрос — **H3** или `<summary>` в `<details>` |
| Schema | **FAQPage** JSON-LD (Наташа), вопросы = видимый текст на странице |

Темы FAQ (покрыть по возможности все):

- стоимость и сроки;
- интеграции (CRM, сайт, мессенджеры);
- риски и 152-ФЗ / персональные данные;
- отличие от чат-бота / менеджера;
- метрики и KPI после внедрения;
- что нужно от клиента для старта;
- поддержка и обучение;
- отраслевые нюансы (если есть в теме).

## Связка ролей

| Роль | Действие |
| --- | --- |
| **Коля** | Оглавление H2/H3 по Маслоу + список **12–18** FAQ-вопросов (черновики) с привязкой к ключам |
| **Артём** | Факты и кейсы; ссылки только в блоке «не для лонгрида» |
| **Женя** | Пишет по лестнице Маслоу; кейсы без URL; FAQ полный |
| **Наташа** | Секция `#faq`, класс `nero-ai-faq`, FAQPage JSON-LD |
| **Макс** | Проверка: нет ссылок в кейсах; FAQ ≥10; schema FAQ |
