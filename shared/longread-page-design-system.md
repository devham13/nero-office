# Дизайн-система лонгрида Configured WordPress Theme (эталон без hero)

**Готовый код в плагине (не нужно открывать сайт за CSS):**

- **`shared/longread-page-design-reference.css`** — полный эталонный CSS из страницы Метрики (классы `ym-*`, обёртка `.metrika-skill-page`, reveal, медиазапросы). Копируй в тему или в `<style>`, замени имя обёртки на свой slug.
- **`shared/longread-page-reveal.js`** — скрипт `IntersectionObserver` для `.reveal` / `.reveal-left` / `.reveal-right` / `.reveal-scale`.

**Эталон в браузере:** [${PUBLIC_SITE_HOST}/metrika-skill/](https://${PUBLIC_SITE_HOST}/metrika-skill/)  
**Исходник темы:** `page-yandex-metrika-skill.php` (тот же CSS, что в `longread-page-design-reference.css`).

## Роль Наташи относительно hero

- **Hero с Canvas** делает **Алина** — готовый блок **вставляется первым** без перерисовки «под метрику».
- Наташа проектирует **всё, что ниже hero**, и общую оболочку страницы: токены, типографика, секции, карточки, оглавление, FAQ, код-блоки, анимацию появления при скролле — **в этом же визуальном языке**, что лонгрид на странице Метрики (сетка, отступы, тени, радиусы).

## Обёртка страницы

- Корневой класс в эталоне: **`metrika-skill-page`** — для новой темы замени на **`{slug}-page`** (например `content-factory-page`), сохрав стили с тем же каскадом: `background`, `font-family: Inter`, `padding-bottom`, `overflow-x: hidden`.
- **Явные цвета заголовков/текста** внутри обёртки (`!important` при необходимости), чтобы не ломал глобальная тёмная тема сайта — как в эталоне.

## CSS-переменные (`:root`)

Подставляй акценты под тему страницы; структура имён сохраняется:

| Переменная | Эталон (Метрика) | Назначение |
|------------|------------------|------------|
| `--ym-bg` | `#f8fafc` | Фон страницы |
| `--ym-surface` | `#ffffff` | Карточки, подложки |
| `--ym-text` | `#334155` | Основной текст |
| `--ym-heading` | `#0f172a` | Заголовки |
| `--ym-border` | `#e2e8f0` | Границы |
| `--ym-primary` | `#ff0000` | Главный CTA / акцент темы |
| `--ym-accent` | `#3b82f6` | Вторичный акцент, ссылки |
| `--ym-code-bg` | `#0f172a` | Фон «терминала» |
| `--ym-code-text` | `#38bdf8` | Акцент в коде |
| `--ym-success` | `#10b981` | Тренды, галочки (задать в `:root`, в эталоне используется в CSS) |
| `--ym-shadow-sm` / `--ym-shadow` / `--ym-shadow-lg` | см. файл | Тени |

На **другой** странице меняй в первую очередь **`--ym-primary`** и при необходимости **`--ym-accent`**, остальное можно оставить для единообразия офиса.

## Анимация при скролле

- Классы: **`.reveal`**, **`.reveal-left`**, **`.reveal-right`**, **`.reveal-scale`** + модификаторы **`.delay-100` … `.delay-500`**.
- Скрипт внизу страницы: `IntersectionObserver` на эти классы, при входе в вьюпорт — класс **`.active`** (см. конец `page-yandex-metrika-skill.php`).

## Контейнер и секции

- **`.ym-container`** — `max-width: 1300px`, центрирование, боковые отступы.
- **`.ym-section`** — вертикальные отступы секций (`padding: 100px 0` в эталоне).
- **`.ym-section-alt`** — чередование фона: белая подложка + бордеры сверху/снизу.
- **`.ym-section-title`** / **`.ym-section-subtitle`** — заголовок H2 блока и подзаголовок по центру.

## Паттерны контента (ниже hero)

### Введение сразу после hero («второй блок» страницы)

Первая секция контента в `<main>` после hero — **не** центрированная «простыня» из абзацев.

| Элемент | Правило |
|--------|---------|
| Выравнивание | Текст вступления — **по левому краю** (`text-align: left`, при конфликте с темой — `!important` на блоке и абзацах). |
| Композиция | **Две колонки** (desktop): слева лид + подводка; справа мини-терминал / схема / KPI-чипы. Акцентная градиентная полоска — у **левого** края текстовой колонки. |
| Оглавление | **`.ym-toc`** — **ниже** блока введения, в отдельной обёртке, обычно **по центру** экрана. |
| Техника | Все стили — в `<style>` страницы с префиксом slug (например `.nero-network-intro-grid`, `.nero-network-intro-text`). **Без Tailwind-only** классов без дублирования в CSS. |
| Адаптив | Одна колонка; текст сверху, декор снизу; текст **слева**. |

Эталон реализации на проде: [${PUBLIC_SITE_HOST}/nero-network-office-page-cursor-plugin/](https://${PUBLIC_SITE_HOST}/nero-network-office-page-cursor-plugin/) (блок «Снимок пайплайна» + лид слева).

**Удержание после hero (в целом):** не начинать лонгрид с длинной «простыни» узкой колонки по центру без визуального якоря. Сразу после блока введения полезны **`.ym-toc`**, **split**-вступление (текст + карточка/метрика), **bento** или чередование **`.ym-section` / `.ym-section-alt`**. Визуальный блок **Бориса** (Canvas) — оформлять **сплитом или сеткой** с подписью, а не одним квадратом по центру (см. skill **animator-boris**).

1. **Оглавление (якоря):** **`.ym-toc`** — flex-wrap, pill-кнопки со ссылками на `#id` секций.
2. **Крупное изображение под hero (опционально):** **`.ym-hero-image-wrap`** — 16/9, скругление, тень, градиентный оверлей снизу.
3. **Bento / метрики:** **`.ym-bento-grid`**, карточки **`.ym-bento-card`**, модификаторы **`.ym-bento-main`**, **`.ym-bento-stat`**, **`.ym-bento-wide`**; статистика **`.ym-stat-value`**, **`.ym-stat-label`**, **`.ym-stat-trend`**.
4. **Сетка возможностей:** **`.ym-grid-3`**, **`.ym-grid-2`**, карточки **`.ym-card`** + **`.ym-card-icon`**.
5. **«Терминал» Mac:** **`.ym-mac-window`**, **`.ym-mac-header`**, точки **`.ym-mac-dot`**, тело **`.ym-mac-body`**, **`.ym-command`**, **`.ym-comment`**.
6. **Таймлайн шагов:** **`.ym-timeline`**, **`.ym-step`**, **`.ym-step-num`**, **`.ym-step-content`**.
7. **Карточки промптов:** **`.ym-prompt-card`**, **`.ym-prompt-text`**, **`.ym-prompt-result`**.
8. **FAQ:** сетка **`.ym-faq-layout`** (сайдбар + контент), **`.ym-faq-sidebar`** sticky, список якорей **`.ym-faq-list`**, блоки ответов **`.ym-faq-item`**.
9. **Связанные материалы:** **`.ym-related-block`**, **`.ym-related-grid`**, **`.ym-related-card`**.
10. **Мета статьи:** **`.ym-article-meta`**, **`.ym-author-box`**, **`.ym-date-box`**.
11. **Кнопки:** **`.ym-btn-group`**, **`.ym-btn`**, **`.ym-btn-primary`**, **`.ym-btn-secondary`**.

## Адаптив

- **1024px:** FAQ в одну колонку; bento — 2 колонки.
- **768px:** уменьшить заголовки hero в **чужом** блоке не трогает Алину; секции **`.ym-section`** плотнее; сетки в 1 колонку.

## Что Наташа выдаёт

- Фрагмент **`<style>`** (или список классов для существующего файла темы) с токенами и компонентами **по этому эталону**, с переименованием обёртки под slug.
- **HTML:** сначала **вставка hero от Алины** (как есть), затем секции с разметкой из текста **Коли** (H2/H3, списки, таблицы при необходимости), якоря, карточки, FAQ, при необходимости **JSON-LD** (Article/FAQPage) — по задаче.
- Внизу — **скрипт reveal** как в эталоне.

## Запрет

- Не подменять **дизайн hero Алины** своим **`.ym-hero`** с той же страницы-эталона; hero — только блок Алины.
