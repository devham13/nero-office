Не используй WebSearch и WebFetch для самостоятельного поиска новости дня.

Используй WebSearch/WebFetch только если нужно уточнить факты по уже выбранной теме из Google Таблицы или проверить источник тренда, указанный в таблице.

Используй Wordstat через MCP KV только для проверки, уточнения или расширения ключей из Google Таблицы: ориентир 3–8 вызовов `wordstat_get_top_requests`, если это необходимо.

Сравни кандидатов по скорингу из skill.

Отклони дубли:

* та же тема;
* тот же смысловой угол;
* похожий slug;
* уже опубликованная страница;
* уже выбранная тема в ledger;
* **в Google Таблице в колонке «Ссылка на страницу» уже стоит URL страницы на сайте** (главный признак «тема уже сделана»).

Не предлагай список тем на выбор. Выбери одну тему, которая лучше всего даст трафик и лиды.

## Google Таблица: резервирование и ссылка

| Этап | Кто | Действие в таблице |
| --- | --- | --- |
| Выбор темы | **Кирилл** | Статус `в работе` / `создаётся`; в handoff — номер строки; **не** дублировать эту строку в следующих запусках |
| Публикация | **Юра** | В колонку **«Ссылка на страницу»** — полный публичный URL (`https://…/slug/`) |
| Следующий запуск | **Кирилл** | Строки с заполненной «Ссылкой на страницу» **не рассматривать** |

Если нет доступа на запись в Google Таблицу (только чтение) — в handoff блок `Статус: ❌ БЛОКЕР` с причиной; не выбирай тему без возможности зарезервировать строку.
